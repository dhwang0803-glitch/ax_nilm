# EXP1 Denoise Ablation 결과

## Denoise란?

집 전체 전력 신호(agg)에서 고주파 잡음을 제거하는 전처리.
신호를 주파수 성분으로 분해 → 작은 고주파 성분만 0으로 깎아냄 → 다시 합성.

30Hz 전력 데이터에 섞인 측정 노이즈를 걷어내 모델이 노이즈 패턴을 학습하지 않도록 하려는 목적이었으나,
인덕션·전기밥솥처럼 **전원 켤 때 순간적으로 전력이 튀는 패턴(transient)** 도 고주파 성분이라 함께 깎여버렸다.

## 해석

### 왜 denoise=OFF가 이겼나

wavelet denoising(db4, level=1)이 고주파 성분을 제거하면서 **빠른 전력 변화 패턴을 가진 기기의 transient 특징이 손상**됐다. level=1로 최소한만 제거했음에도 인덕션(−0.172), 전기밥솥(−0.069)에서 유의미한 성능 저하가 나타남.

### F1_cls는 왜 ON에서 높았나 (+0.076)

denoising으로 steady-state 패턴이 깔끔해져 BCE 분류 헤드 학습이 쉬워진 것. 그러나 transient 구간 식별력 손실이 더 크게 작용해 전체 성능은 OFF가 우세함. 분류 헤드와 회귀 헤드 간 트레이드오프.

### Always-on 기기가 전부 동등한 이유

냉장고·공기청정기 등은 transient가 거의 없어 denoising 영향 자체가 없음.

## EXP2 이후 시사점

- `denoise=OFF`를 기본값으로 고정
- **인덕션, 전기밥솥** F1이 전반적으로 낮음(0.2~0.5대). denoising이 아닌 다른 접근 필요:
  - 이벤트 윈도우 전략 조정 (전환점 주변 윈도우 밀도 증가)
  - 클래스 가중치(pos_weight) 재조정
  - 데이터 보강 (해당 기기 보유 house 추가)
