# 저주파(30Hz) 전력 데이터로 TDA 상태 모니터링이 가능한 이유

> 검증 근거: `tda_validation_result.md` (2026-05-14, AI Hub 가전 10종)

---

## 주장

> **단일 분전반 계량기의 30Hz 전력(W) 데이터만으로,
> 진폭 기반 방법으로는 구분 불가능한 가전 동작 모드를 TDA로 식별한다.**

고주파 전류·전압 센서 없이 상태 모니터링이 가능하다는 것이 핵심이다.

---

## 파이프라인 개요

```
ON 구간 전력 신호 (30Hz, W)
        │
        ▼
┌─────────────────────────┐
│  슬라이딩 윈도우         │  512 samples = 17초 @ 30Hz
│  (stride = 1 window)    │  → 가전 1사이클 이상의 컨텍스트 확보
└────────────┬────────────┘
             │
        [Per-segment 정규화]
        (x - min) / (max - min)     → 진폭 제거, 파형 형태만 보존
             │
             ▼
┌─────────────────────────┐
│  Time-delay 임베딩       │  dim=3, lag=10 (= 0.33s)
│  (Takens' theorem)      │  x(t) → [x(t), x(t+lag), x(t+2·lag)]
└────────────┬────────────┘
             │  1D 신호 → 3D 위상 공간의 점군(point cloud)
             ▼
┌─────────────────────────┐
│  ripser H1               │  Vietoris-Rips 필트레이션
│  (Persistent Homology)  │  1차 호몰로지 → 루프 구조 추출
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  Persistence Image       │  20×20 픽셀
│                         │  birth/persistence → 이미지 표현
│                         │  → 클러스터링 입력 벡터
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  K-Means 클러스터링      │  K = silhouette 최대화로 자동 선택
│                         │  (K 범위: 2~5 탐색)
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│  3중 검증               │
│  ① Silhouette Score     │  위상 군집 구조 존재 여부  (기준: ≥ 0.3)
│  ② NMI / ARI vs W분포  │  W 진폭과의 독립성         (기준: NMI < 0.3)
│  ③ 시간 일관성 (run)    │  물리적 상태 반영 여부     (기준: run ≥ 3)
└─────────────────────────┘
```

## 왜 30Hz에서도 작동하는가 — 단계별 이유

### 1. 슬라이딩 윈도우: 가전 동작 주기를 통째로 담는다

고주파 방법이 필요한 이유는 **전기 고조파(250Hz~)**를 포착하기 위해서다.
TDA가 보는 것은 고조파가 아니라 **수 초~수십 초 단위의 동작 패턴**이다.

```
압축기 ON/OFF 주기:  수 초 ~ 수십 초
모터 가속/제동:      수 초
써모스탯 사이클:     수십 초 ~ 수 분

30Hz × 512 samples = 17초  →  이 스케일을 완전히 커버
```

창이 가전의 물리적 사이클 한 주기를 담을 만큼 길기 때문에,
저주파여도 상태 구조 전체가 임베딩 공간에 펼쳐진다.

### 2. Time-delay 임베딩: 1D 신호를 3D 위상 궤적으로 변환

Takens 정리에 의해, `[x(t), x(t+lag), x(t+2·lag)]` 임베딩은
원래 동역학계의 어트랙터를 **위상 동형으로 복원**한다.

```
세탁기 교반 상태:
  30Hz 신호가 0~160W를 격렬하게 진동
  → 3D 임베딩: 원점에서 멀리 퍼진 넓은 루프
  → H1 persistence: 크고 오래 사는 루프 feature

세탁기 대기 상태:
  30Hz 신호가 0~15W 완전 평탄
  → 3D 임베딩: 원점 근처 작은 점군
  → H1 persistence: feature 없음
```

30Hz로도 이 형태 차이는 보존된다. 고주파가 필요한 것은
**전기 고조파 분해**이지, **이 궤적 형태의 구분**이 아니기 때문이다.

### 3. Per-segment 정규화: 진폭을 지우고 형태만 남긴다

```python
x_norm = (x - x.min()) / (x.max() - x.min())
```

이 정규화가 핵심이다. W 절댓값을 제거하면:
- 임베딩 공간에서 점군의 **절대 위치(진폭)**가 아닌 **형태(곡률, 루프)**만 남음
- W가 거의 같은 두 상태도, 파형 형태가 다르면 H1 구조가 달라짐

검증 결과에서 NMI < 0.3 (8/10 가전) → TDA 클러스터가 W 분포와 독립적임을 직접 증명.

### 4. ripser H1 → Persistence Image: 노이즈를 자동으로 무시한다

Persistent Homology의 핵심 성질:
- **단수명 feature** (생성 직후 소멸) → 노이즈로 처리, 무시
- **장수명 feature** (오래 지속) → 실제 위상 구조, 보존

30Hz 신호에는 고주파 노이즈가 없고, 이미 저역통과 필터링된 신호다.
H1이 추출하는 루프 구조의 **persistence가 오히려 더 선명**해진다.

Persistence Image (20×20) 변환으로 birth/persistence 평면을
고정 크기 벡터로 만들어 K-Means 입력으로 사용한다.

### 5. K-Means + Silhouette 자동 선택: W-range보다 정확한 K를 찾는다

W-range 방법은 진폭 임계값으로 K를 미리 고정한다.
TDA는 silhouette 최대화로 K를 데이터에서 자동 도출한다.

```
에어컨:     W-range K=3  →  TDA K=2  (W-range가 위상적으로 같은 모드를 과분할)
세탁기:     W-range K=2  →  TDA K=3  (W-range가 놓친 하위 모드를 TDA가 추가 발견)
```

저주파 데이터에서도 위상 구조 기반 K 선택이
진폭 기반 K 선택보다 실제 동작 모드에 더 가깝다.

## 검증 수치 요약

| 지표 | 기준 | 결과 | 해석 |
|------|------|------|------|
| Silhouette ≥ 0.3 | TDA가 군집 구조 발견 | 10/10 가전 통과 | 저주파에서도 위상 구조 존재 |
| NMI < 0.3 | W와 독립적 구조 | 8/10 가전 통과 | 진폭 정보 없이도 분리 가능 |
| run ≥ 3 | 물리 상태 반영 | 7/10 가전 통과 | 노이즈 분류 아님, 실제 상태 전환 |

### 핵심 사례 — W 차이 극소 구분

| 사례 | W 차이 | W-range | TDA |
|------|--------|---------|-----|
| 세탁기 교반 vs 대기 | **0.2W** | ❌ 불가 | ✅ 루프 구조 차이로 분리 |
| 전기장판 정상 vs 보상 히팅 | **2.8W** | ❌ 불가 | ✅ 추가 펄스 패턴으로 분리 |
| 제습기 안정 vs 전환 혼재 | 동일 밴드 (22~24W) | ❌ 불가 | ✅ 파형 안정성으로 분리 |

## 한 줄 결론

> 가전 상태 전환은 수 초~수십 초 스케일의 동역학이다.
> Time-delay 임베딩이 이 동역학의 위상 궤적을 복원하고,
> ripser H1이 그 구조를 추출하기 때문에 —
> **주파수 해상도가 아닌 시간 패턴 해상도가 필요하고, 30Hz는 그것을 충분히 제공한다.**
